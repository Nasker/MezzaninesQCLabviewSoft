Supported Model(s): MSO8104A, MSO8064A, DSO8104A, DSO8064A, DSO80204B, DSO80304B, DSO80404B, DSO80604B, DSO80804B, DSO81004B, DSO81204B, DSO81304B, DSO80404A, DSO81004A, DSO81204A, DSO81304A, DSA80204B, DSA80304B, DSA80404B, DSA80604B, DSA80804B, DSA81004B, DSA81204B, DSA81304B

2. Required Software

Some software components need to be installed before using this instrument driver. The minimum versions of these components are listed below, and can be downloaded from the Download Site.

VISA 5.0 or later

Refer to the LabVIEW Help for more information about software requirements. You access the LabVIEW Help by selecting Help»Search the LabVIEW Help.

3. Known Issues

To report issues or provide feedback about this instrument driver, please send an email to instrument.drivers@ni.com.

  • Running VISA for a prolonged period of time may cause main instrument application to crash.

  • Manual says that enabling mark testing will avoid the "settings conflict" error when calling other mask commands. However, this does not appear to be the case. After enabling mark testing through the "Configure Mark Test" VI, calling "Configure Mask Scaling" still causes the "settings conflict" error.

  • RJDJ Extraction Measurement is only supported for Agilent 80,000 series.

  • "Fetch Measurement(Data Rate-Unit Interval)" and "Configure Measurement Trend Setting" VIs work for Agilent 8000 series but still returns mulitple errors for reasons unknown.
